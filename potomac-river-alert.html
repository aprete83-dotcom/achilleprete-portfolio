Overview

Turning NOAA river gauge data into a useful decision tool.

This project is a working prototype for an automated river monitoring system designed to help kayakers decide when Potomac River conditions are within a preferred paddling range.

As a kayaker, checking river levels is part of the planning process. Instead of manually visiting the gauge website multiple times, this page retrieves the latest available river level from NOAA’s National Water Prediction Service API when the page loads.

The system compares the current level with a user-defined minimum and maximum range. This demonstrates how public data, automation, and simple decision logic can support faster and better planning.

Problem

River levels change quickly, and manual checking is inefficient.

River levels can change throughout the day depending on rain, upstream conditions, dam releases, and seasonal flow. For kayakers, timing matters. A level that is too low may not be worth the trip, while a level that is too high may create unsafe conditions.

Manually checking the same website repeatedly is inefficient, especially when the user only cares about a specific range of values.

  • River conditions can change throughout the day.
  • Manual checking takes time and can be easy to forget.
  • The user may only care when the level is inside a specific kayaking range.
  • A simple dashboard or alert can help the user make faster decisions.

Solution

Live gauge check with threshold logic

The system checks NOAA’s NWPS API for the BRKM2 gauge, reads the latest available Potomac River level, compares it with a preferred kayaking range, and displays the current status.

01

Call NOAA API

JavaScript requests the observed stage/flow data for gauge BRKM2.

02

Read Latest Level

The page extracts the latest available river level value from the API response.

03

Compare Range

The level is compared with the preferred minimum and maximum range.

04

Display Status

The dashboard shows whether the river is below, within, or above the selected range.

Why It Matters

A small system that turns public data into action.

This project demonstrates how a simple information system can turn live public data into a useful personal decision-making tool.

The system does not need to be large or complex to be valuable. By automating a repeated task, it saves time, reduces manual checking, and gives the user useful information at the right moment.

Reflection

What this project shows

This project shows the ability to identify a real-world need, connect to an external data source, process live data, apply decision logic, and display a result through a user-friendly interface.

The same logic could be applied to facilities monitoring, equipment alerts, inventory thresholds, weather notifications, or any process where users need to know when data reaches a specific condition.
