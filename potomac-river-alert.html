Overview

Turning live river data into a useful alert system.

This project is a concept for an automated river monitoring system designed to help kayakers decide when conditions are within a preferred paddling range.

As a kayaker, checking river levels is part of the planning process. Instead of manually visiting the same gauge website multiple times, this system would automatically check the Potomac River level, compare the current reading against a user-defined range, and send an email alert when the level is suitable for kayaking.

The goal of the project is to show how public data, automation, and simple notification logic can support faster and better decision-making.

Problem

River levels can change quickly, and manual checking is inefficient.

River levels can change quickly depending on rain, dam releases, and upstream conditions. For kayakers, timing matters. A level that is too low may not be worth the trip, while a level that is too high may create unsafe conditions.

Manually checking the same website repeatedly is inefficient, especially when the user is only interested in a specific range of values.

  • River conditions can change throughout the day.
  • Manual checking takes time and can be easy to forget.
  • The user may only care when the level is inside a specific range.
  • A simple alert can help the user make faster decisions.

Solution

Automated gauge check and email notification logic

The proposed system checks an official river gauge source, reads the current river level, compares it with a preferred kayaking range, and sends an email when the level is within range.

01

Check Gauge

The system checks the river gauge data source on a schedule.

02

Read Level

The current river level is retrieved, cleaned, and stored.

03

Compare Range

The level is compared with the preferred minimum and maximum range.

04

Send Alert

An email is sent when the level is within the selected range.

Why It Matters

A small system that turns data into action.

This project demonstrates how a simple information system can turn live public data into a useful personal decision-making tool.

The system does not need to be large or complex to be valuable. By automating a repeated task, it saves time, reduces manual checking, and gives the user useful information at the right moment.

Reflection

What this project shows

This project shows the ability to identify a real-world need, connect to an external data source, process live data, apply decision logic, and trigger an automated action.

The same logic could be applied to facilities monitoring, equipment alerts, inventory thresholds, weather notifications, or any process where users need to be notified when data reaches a specific condition.
